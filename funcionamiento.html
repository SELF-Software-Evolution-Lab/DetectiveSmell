¿Qué hace el proyecto?

DetectiveSmell analiza minuciosamente el código de tu proyecto en busca de errores comunes y áreas de mejora. Utilizando tecnologías avanzadas como ANTLR4, ts-morph y java-ast, esta extensión identifica "code-smell" en etiquetas de frameworks antes de la ejecución del programa, ayudando a mejorar la calidad del código y la eficiencia del desarrollo. Con soporte para múltiples frameworks y reglas personalizables, DetectiveSmell se integra directamente en Visual Studio Code, proporcionando mensajes de error claros y concisos que indican la ubicación del error y cómo corregirlo. Además, categoriza los errores detectados según su gravedad y la capa a la que pertenecen, permitiendo a los desarrolladores priorizar y planificar sus esfuerzos de refactorización de manera efectiva.

¿Cómo funciona?

Visual Studio Code

  • Para empezar, al hacer click derecho en el proyecto que se quiere analizar, aparecerá una serie de opciones del menú, en donde se debe seleccionar el que dice "Analizar proyecto".
  • Una vez seleccionado el proyecto, se puede dirigir a "settings" para seleccionar el tipo de proyecto, las reglas y las capas que se desea analizar.
  • Una vez analizado el proyecto, la extensión arrojará un reporte en donde se describe en que línea de código se encuentra el error, que regla esta rompiendo y el tipo de severidad.

API REST

Entrada (Tipo POST):

  • La API REST está implementada en Node.js, las solicitudes se realizan utilizando el método POST y se accede a través del endpoint '/analize'.
  • Se envía un JSON como cuerpo de la solicitud, que contiene los siguientes campos: 
    {
	"url": "URL_GIT",
	"token": "Token de autenticación",
	"type": "Tipo de proyecto (spring, nest, angular)"
}

Salida (JSON reporte):

  • La respuesta de la API es un JSON que contiene una capa con diferentes atributos dependiendo del tipo de proyecto analizado.

  • Para proyecto SpringBoot:

    • Capa de persistencia: Información sobre errores relacionados con la persistencia de datos.
    • Capa de lógica: Información sobre errores en la lógica de negocio.
    • Capa de controladores: Información sobre errores en los controladores.

  • Para proyecto NestJS:

    • Capa de persistencia: Información sobre errores relacionados con la persistencia de datos.
    • Capa de lógica: Información sobre errores en la lógica de negocio.
    • Capa de controladores: Información sobre errores en los controladores.
    • Capa de DTO: Información sobre errores en los objetos de transferencia de datos.

  • Para proyecto Angular:

    • Capa de infraestructura: Información sobre errores relacionados con la infraestructura del proyecto.
    • Capa de módulos: Información sobre errores en los módulos del proyecto.
  • Dentro de cada capa, se presentan los siguientes atributos para describir los errores encontrados: 
    {
 "Capa de ": [
		{
			"message": "Indica en qué archivo se presentó el posible error.",
			"severity": "La gravedad del error encontrado.",
			"name": "El título del posible error.",
			"id": "Identificador de la regla.",
			"description": "Descripción del porqué es un error.",
			"example": "Muestra cómo debería corregirse el error.",
			"line": "El número de la línea de código donde se encuentra el error en el archivo.",
			"path": "La ruta relativa del archivo con el error.",
			"absolutePath": "La ruta completa del archivo con el error."
		}
	    ]
}
